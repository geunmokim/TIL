# C++ STL 개념 정리

---

## 1. STL이란?

**Standard Template Library** (표준 템플릿 라이브러리)  
C++에서 제공하는 라이브러리의 일종으로, 프로그래밍에 필요한 **자료구조 및 알고리즘**을 클래스 템플릿화하여 제공한다.

---

## 2. STL 구성요소

### ① 컨테이너
데이터를 저장하는 객체. 자료구조를 구현한 객체.

```
vector  → 배열 기반
list    → 노드 기반
map     → 트리 기반
```

#### 컨테이너 구분 기준

**원소 배치 방식:**

| 종류 | 구조 | 컨테이너 |
|---|---|---|
| 표준 시퀀스 컨테이너 | 선형적 구조 (앞/뒤 판단 가능) | vector, list |
| 표준 연관 컨테이너 | 비선형적 구조 (앞/뒤 판단 불가) | map |

**메모리 저장 방식:**

| 종류 | 방식 | 컨테이너 |
|---|---|---|
| 배열 기반 | 연속된 메모리 사용 | vector |
| 노드 기반 | 메모리 여기저기 저장, 포인터로 연결 | list, map |

---

### ② 알고리즘
컨테이너 내에서 정렬, 삭제, 탐색 등을 해결하는 일반화된 함수 템플릿.

```cpp
#include <algorithm>  // 자주 사용하는 알고리즘 함수들이 여기 있음
```

> 💡 대부분의 알고리즘 함수들은 컨테이너의 멤버가 아닌 **전역**에 정의되어 있다.

---

### ③ 함수 객체
함수 호출 연산자 `()`를 연산자 오버로딩하여 **객체를 함수처럼 사용**하는 것.

```cpp
// 함수 객체 예시
struct Less {
    bool operator()(int a, int b) {
        return a < b;  // 객체인데 함수처럼 호출 가능
    }
};
```

---

### ④ 반복자 (Iterator)
**STL의 핵심!**

컨테이너마다 원소의 저장 방식이 달라서 순회 방법도 달라지는데,  
반복자를 이용하면 **어떤 컨테이너든 동일한 방법으로 순회**할 수 있다.

```cpp
// vector든 list든 map이든 동일한 방식으로 순회
for (auto it = container.begin(); it != container.end(); ++it)
    cout << *it << endl;
```

> ⚠️ 사용 방법이 포인터와 비슷하지만 **포인터가 아닌 객체**다!

## 3. 컨테이너 상세 비교

### vector (동적 배열)

- 동적 배열 기반 → **인덱스 접근 가능**
- 앞에서 삽입 불가, **뒤에서만 삽입**
- 중간 삽입/삭제 시 해당 인덱스 이후 원소를 전부 이동해야 함 (선형 시간)
- 맨 끝 삽입/삭제는 빠름 (상수 시간)
- 크기를 초과하면 **재할당 및 복사** 발생
- 원소 삭제해도 배열 크기는 줄어들지 않음

```
장점: 탐색이 빠르다
단점: 삽입/삭제가 느리다
```

---

### list (연결 리스트)

- 더블 링크드 리스트로 구현
- 노드 기반 → **인덱스 접근 불가**
- 원하는 원소를 찾으려면 처음부터 순차적으로 순회해야 함
- 중간 삽입/삭제 시 포인터만 수정하면 됨 → vector보다 훨씬 빠름
- 메모리 제한 없이 런타임 중 계속 노드 추가 가능

```
장점: 삽입/삭제가 빠르다
단점: 탐색이 느리다
```

---

### map (트리)

- key-value 쌍으로 원소 저장
- 삽입 시 **key값 기준으로 자동 정렬**
- 정렬되기 때문에 반복자를 통한 순회 가능
- 노드 기반 컨테이너 중 유일하게 **key값으로 임의 접근 가능**
- **중복 key 불가**
- 리소스 탐색 용도로 많이 사용

```
장점: key-value 관리, key로 빠른 탐색
단점: 삽입/삭제/탐색 속도가 vector보다 느림 (트리 탐색)
```

---

## 4. 한눈에 비교

| | vector | list | map |
|---|---|---|---|
| 기반 | 동적 배열 | 연결 리스트 | Red-Black Tree |
| 메모리 | 연속 | 분산 (포인터 연결) | 분산 (포인터 연결) |
| 인덱스 접근 | ✅ 가능 | ❌ 불가 | ✅ key로 가능 |
| 탐색 속도 | ✅ 빠름 | ❌ 느림 | 🔺 중간 |
| 삽입/삭제 속도 | ❌ 느림 | ✅ 빠름 | 🔺 중간 |
| 자동 정렬 | ❌ | ❌ | ✅ (key 기준) |
| 중복 허용 | ✅ | ✅ | ❌ (key 중복 불가) |

---

## 5. 컨테이너 선택 기준

| 상황 | 선택 |
|---|---|
| 데이터를 많이 **조회**할 때 | `vector` |
| 데이터를 많이 **삽입/삭제**할 때 | `list` |
| 데이터를 **쌍(key-value)으로 관리**할 때 | `map` |
