# C++ STL map 완전 정리

---

## 1. map이란?

pair(구조체)들을 key 기준으로 정렬해서 저장하는 컨테이너.  
내부적으로 **Red-Black Tree** 형태로 저장되며, key는 **중복 불가**.

```cpp
#include <map>  // 헤더 추가 필수

map<int, int> Mymap;  // <key 자료형, value 자료형>
```

---

## 2. pair 구조체

key와 value를 **하나씩만** 저장하는 구조체.  
`first` = key, `second` = value

```cpp
// pair의 실제 내부 구조 (단순화 버전)
template <typename T1, typename T2>
struct pair {
    T1 first;   // key
    T2 second;  // value
};

pair<int, int> Mypair(1, 100);
// Mypair.first  = 1   (key)
// Mypair.second = 100 (value)

Mypair.first  = 2;   // 값 변경 가능
Mypair.second = 200;
```

> ⚠️ pair 자체는 하나의 값만 가진다. 데이터가 쌓이는 곳은 map이다!

```
Mymap
┌──────────────────────┐
│ first:1, second:100  │  ← 1번째 insert로 복사된 것
├──────────────────────┤
│ first:2, second:200  │  ← 2번째 insert로 복사된 것
└──────────────────────┘
```

---

## 3. 원소 삽입 방법

map은 key 기준으로 자동 정렬되므로 `push_back` 대신 `insert` / `emplace` 사용.

```cpp
map<int, int> Mymap;

Mymap[0] = 0;                                    // 방법1: operator[]
Mymap.insert(pair<int, int>(1, 100));            // 방법2: pair 직접 생성
Mymap.insert(make_pair(2, 200));                 // 방법3: make_pair (자료형 자동 추론)
Mymap.insert(map<int, int>::value_type(3, 300)); // 방법4: value_type
Mymap.emplace(4, 400);                           // 방법5: emplace (가장 간결)
Mymap.insert({ 5, 500 });                        // 방법6: 중괄호 초기화
```

---

## 4. insert() vs operator[]

```cpp
Mymap.insert({1, 100});  // key 1이 이미 있으면 → 무시
Mymap[1] = 100;          // key 1이 이미 있으면 → 덮어씀
```

| 방법 | key 중복 시 |
|---|---|
| `insert()` | ❌ 무시 |
| `Mymap[key] = value` | ✅ 덮어씀 |

> 💡 insert()는 복사본을 저장한다.  
> Mypair를 수정해도 이미 map에 들어간 값은 영향받지 않는다.

## 5. 반복자 (Iterator)

map의 반복자는 **양방향 접근 반복자**.

- `begin()` → 가장 작은 key를 가리킴
- `end()` → 마지막 key의 다음 (실제 데이터 아님)

```cpp
auto iter = Mymap.begin();  // key:1 가리킴
++iter;                      // key:2 가리킴
++iter;                      // key:3 가리킴

iter->first;   // key 접근
iter->second;  // value 접근

// 아래는 불가능!
iter[2];    // ❌
iter += 2;  // ❌
```

## 6. 중간 삽입 / 삭제

### 중간 삽입

map은 자동 정렬되므로 중간 삽입의 의미는 없다.  
iter는 **탐색 힌트**로만 활용된다.

```cpp
Mymap.insert({9, 900});          // 트리 처음부터 탐색
Mymap.insert(iter, {9, 900});    // iter 근처부터 탐색 → 약간 더 빠를 수 있음
// 결과는 동일, 힌트가 틀려도 오류 없음
```

### 중간 삭제

```cpp
Mymap.erase(iter);  // iter가 가리키는 원소 삭제

// ⚠️ 삭제 후 iter는 무효화됨!
// iter++;  // 위험!

// 삭제 후 iter를 계속 쓰고 싶으면:
iter = Mymap.erase(iter);  // erase가 다음 iter를 반환해줌
```

## 7. 원소 정렬

기본값은 **오름차순** 정렬.  
내림차순으로 바꾸려면 `greater<int>` 사용.

```cpp
map<int, int, greater<int>> Mymap;  // 내림차순
Mymap.insert({ 5, 500 });
Mymap.insert({ 3, 300 });
Mymap.insert({ 1, 100 });

for (auto& pair : Mymap)
    cout << pair.first << ", " << pair.second << endl;
// 출력: 5 → 3 → 1 순서
```

## 8. 원소 탐색

### operator[] 탐색의 문제점

> ⚠️ 없는 key를 탐색하면 디폴트값(0)으로 원소가 자동 추가된다!

```cpp
cout << Mymap[9] << endl;
// key 9가 없으면 {9, 0}이 자동으로 추가됨 → 의도치 않은 원소 삽입!
```

### find() 멤버 함수

- 탐색 성공 → 해당 위치의 반복자 반환
- 탐색 실패 → `end()` 반환

```cpp
auto iter = Mymap.find(9);

if (iter != Mymap.end())  // end()와 다르면 탐색 성공
    cout << iter->first << ", " << iter->second << endl;
else
    cout << "탐색 실패!" << endl;
```

### find()의 주의사항

`const char*`를 key로 사용하면 **주소값을 비교**하기 때문에 탐색이 실패할 수 있다!

```cpp
// ❌ 잘못된 방법 - 주소값 비교로 탐색 실패 가능
map<const char*, int> Mymap;
Mymap.emplace("AAA", 100);
Mymap.find("AAA");  // "AAA"의 주소가 달라서 실패할 수 있음!

// ✅ 올바른 방법 - 문자열 자체를 비교
map<string, int> Mymap;
Mymap.emplace("AAA", 100);
Mymap.find("AAA");  // 정상 동작!
```

## 9. find_if() 알고리즘 함수

`find()`는 key로만 탐색하지만, `find_if()`는 **내가 정한 조건**으로 탐색한다.

| | 탐색 기준 | 속도 |
|---|---|---|
| `find()` | key값으로 탐색 | ✅ 빠름 (트리 탐색) |
| `find_if()` | 내가 정한 조건 | 🔺 느림 (처음부터 순회) |

> key로 찾을 수 있으면 `find()` 사용,  
> value나 복잡한 조건이 필요할 때만 `find_if()` 사용!

### find_if() 사용법 (람다식 권장)

```cpp
map<int, int> Mymap;
Mymap.emplace(1, 100);
Mymap.emplace(2, 200);
Mymap.emplace(3, 300);

// value가 200인 것을 찾고 싶을 때
auto iter = find_if(Mymap.begin(), Mymap.end(),
    [](pair<int,int> p) {
        return p.second == 200;
    });

if (iter != Mymap.end())
    cout << iter->first << ", " << iter->second << endl;
```

### 람다식 구조

```
[캡처절] (매개변수) { 함수 본문 }
```

| 캡처절 | 의미 |
|---|---|
| `[]` | 외부 변수 사용 안함 |
| `[target]` | target 변수만 가져옴 (읽기 전용) |
| `[&target]` | target을 참조로 가져옴 (수정 가능) |
| `[&]` | 외부 변수 전부 참조로 가져옴 |
| `[=]` | 외부 변수 전부 복사로 가져옴 |

### 람다식 활용 예시

```cpp
int target = 200;

// ① 외부 변수로 탐색
find_if(Mymap.begin(), Mymap.end(),
    [target](pair<int,int> p) { return p.second == target; });

// ② 범위 조건
find_if(Mymap.begin(), Mymap.end(),
    [](pair<int,int> p) { return p.second > 100 && p.second < 300; });

// ③ key와 value 동시 조건
find_if(Mymap.begin(), Mymap.end(),
    [](pair<int,int> p) { return p.first == 2 && p.second == 200; });

// ④ 외부 변수 수정 ([&] 사용)
int count = 0;
find_if(Mymap.begin(), Mymap.end(),
    [&count](pair<int,int> p) {
        count++;  // 실행될 때마다 count 증가
        return p.second == 200;
    });
```

## 10. 컨테이너 선택 기준

| 컨테이너 | 언제 사용? |
|---|---|
| `vector` | 데이터를 많이 **조회**할 때 |
| `list` | 데이터를 많이 **삽입/삭제**할 때 |
| `map` | 데이터를 **쌍(key-value)으로 관리**할 때 |
