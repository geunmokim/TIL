# C++ STL vector 완전 정리

---

## 1. vector란?

**동적 배열** 기반 컨테이너. 연속된 메모리에 원소를 저장하며 인덱스로 빠르게 접근할 수 있다.

```cpp
#include <vector>  // 헤더 추가 필수

vector<int> vecInt;
```

---

## 2. 원소 삽입

### push_back()
맨 뒤에 원소 추가.

```cpp
vecInt.push_back(1);  // [1]
vecInt.push_back(2);  // [1, 2]
vecInt.push_back(3);  // [1, 2, 3]
vecInt.push_back(4);  // [1, 2, 3, 4]
vecInt.push_back(5);  // [1, 2, 3, 4, 5]
```

### emplace_back()
맨 뒤에 원소를 **직접 생성**하여 추가. 복사/이동 없이 생성하므로 push_back보다 효율적.

```cpp
vector<string> v;
v.emplace_back("Hello");  // 객체를 직접 생성 (복사/이동 없음)
```

### insert()
특정 위치에 원소 삽입. 중간 삽입이 많다면 list 사용을 권장.

```cpp
vector<int> v = {1, 2, 4, 5};
v.insert(v.begin() + 2, 3);  // 인덱스 2 위치에 3 삽입

// 결과: [1, 2, 3, 4, 5]
// 삽입 후 뒤의 원소들이 한 칸씩 밀려남
```

> ⚠️ vector는 중간 삽입 시 뒤의 원소들을 전부 복사/이동해야 해서 비효율적이다.  
> 중간 삽입/삭제가 많다면 `list`를 사용하자!

## 3. 원소 삭제

### pop_back()
맨 뒤 원소 1개 삭제.

```cpp
vecInt.pop_back();  // [1, 2, 3, 4]
vecInt.pop_back();  // [1, 2, 3]
```

### erase()
특정 위치 또는 구간 삭제.

```cpp
vector<int> v = {1, 2, 3, 4, 5};

// 특정 위치 삭제
v.erase(v.begin() + 2);  // 3 삭제 → [1, 2, 4, 5]

// 반복자로 삭제
auto it = v.begin() + 2;
v.erase(it);

// 구간 삭제 (begin+1 부터 begin+4 전까지)
v.erase(v.begin() + 1, v.begin() + 4);  // 2, 3, 4 삭제 → [1, 5]
```

### clear()
모든 원소 삭제. (size는 0이 되지만 capacity는 유지됨)

```cpp
v.clear();
cout << v.size();  // 0
```

### erase-remove 패턴
특정 값을 가진 원소 전부 삭제.

```cpp
vector<int> v = {1, 3, 2, 3, 4, 3};

v.erase(remove(v.begin(), v.end(), 3), v.end());
// 결과: [1, 2, 4]

// remove()  → 삭제할 원소를 뒤로 몰아놓음 (실제 삭제 X)
// erase()   → 실제로 크기를 줄임
```

### 삭제 함수 정리

| 함수 | 기능 |
|---|---|
| `pop_back()` | 맨 뒤 원소 1개 삭제 |
| `erase(pos)` | 특정 위치 원소 삭제 |
| `erase(first, last)` | 구간 삭제 |
| `clear()` | 전체 삭제 |
| `erase(remove(...))` | 특정 값 모두 삭제 |

> 💡 실무에서는 `pop_back()`, `erase()`, `erase-remove` 패턴을 많이 쓴다.

## 4. 주요 함수

### size()
현재 원소의 개수 반환.

```cpp
vecInt.push_back(1);  // [1, 2, 3, 4, 5]
vecInt.push_back(2);
vecInt.push_back(3);
vecInt.push_back(4);
vecInt.push_back(5);
vecInt.pop_back();    // [1, 2, 3, 4]
vecInt.pop_back();    // [1, 2, 3]

cout << vecInt.size();  // 3
```

### empty()
원소가 없으면 `true`, 있으면 `false` 반환.

```cpp
vecInt.clear();

if (vecInt.empty())
    cout << "원소가 비어있다!" << endl;  // 출력됨
else
    cout << "원소가 비어있지 않다!" << endl;
```

### front() / back()
```cpp
cout << vecInt.front() << endl;  // 맨 앞 원소 출력
cout << vecInt.back() << endl;   // 맨 뒤 원소 출력
```

### capacity()
현재 할당된 메모리 공간의 크기 반환.

```cpp
vecInt.push_back(1);  // capacity: 1
vecInt.push_back(2);  // capacity: 2
vecInt.push_back(3);  // capacity: 4 (재할당!)
vecInt.push_back(4);  // capacity: 4
vecInt.push_back(5);  // capacity: 8 (재할당!)

cout << vecInt.capacity();  // 8
```

> 💡 왜 5개 넣었는데 capacity가 8인가?

| push_back 후 원소 수 | capacity |
|---|---|
| 1 | 1 |
| 2 | 2 |
| 3 | 4 (재할당) |
| 4 | 4 |
| 5 | 8 (재할당) |

메모리 크기를 초과하는 삽입이 발생하면 **기존 크기의 2배**로 재할당된다.  
`clear()`를 해도 size는 0이 되지만 capacity는 유지된다!

## 5. 메모리 예약

### 방법1 - 생성자
생성자 인자만큼 메모리를 설정하지만, **원소도 채워 넣는다**.  
사용 전에 `clear()`로 비우고 사용하자.

```cpp
vector<int> vecInt(10);  // 원소 10개로 초기화됨
vecInt.clear();          // 원소는 비우되 capacity(10)는 유지

cout << vecInt.size();      // 0
cout << vecInt.capacity();  // 10
```

### 방법2 - reserve()
원소 없이 **메모리만** 미리 예약한다.

```cpp
vector<int> vecInt;
vecInt.reserve(10);  // 원소 없이 메모리만 예약

cout << vecInt.size();      // 0
cout << vecInt.capacity();  // 10
```

## 6. 반복자 (Iterator)

반복자는 포인터처럼 사용하지만 **포인터가 아닌 객체**다.  
컨테이너마다 다른 저장 방식을 획일화하기 위해 만들어졌다.

vector는 **임의 접근 반복자**를 사용한다.

```cpp
vector<int>::iterator iter = vecInt.begin();
// 또는
auto iter = vecInt.begin();

iter[3];   // ✅ 가능 (임의 접근)
iter += 3; // ✅ 가능 (임의 접근)
++iter;    // ✅ 가능
--iter;    // ✅ 가능
```

### begin() / end()

```cpp
vector<int>::iterator iter_begin = vecInt.begin();  // 첫 번째 원소
vector<int>::iterator iter_end   = vecInt.end();    // 마지막 다음 위치

// 반복자로 원소에 10씩 더하기
vector<int> vec = {1, 2, 3};
for (; iter_begin != iter_end; ++iter_begin)
    *iter_begin += 10;
// 결과: {11, 12, 13}
```

### 반복자 종류

| 종류 | 사용 컨테이너 | 가능한 연산 |
|---|---|---|
| 임의 접근 반복자 | vector, deque | `++`, `--`, `[]`, `+=`, `-=` |
| 양방향 접근 반복자 | list, map, set | `++`, `--` |
| 순방향 접근 반복자 | forward_list | `++` |

## 7. 반복자 무효화

vector에서 삽입/삭제 시 메모리를 통째로 재할당하면 **기존 반복자가 무효화**된다.

```
vecInt 상태: [1][2][3][4][5]  // 주소 100~120
iter가 3을 가리키는 중 (주소 108)

push_back으로 원소 추가 → 메모리 부족 → 통째로 이동!
새 주소: 200~224

iter는 아직 주소 108을 가리킴 → 쓰레기값 또는 크래시!
```

> ⚠️ 이 문제를 디버깅하기 어려운 이유는 오류 없이 정상 실행되는 것처럼 보이지만  
> 모든 원소를 출력하지 못하는 문제가 발생하기 때문이다.  
> 이를 방지하기 위해 컴파일러가 반복자 무효화 오류를 발생시켜준다.

## 8. 알고리즘 함수

### sort()
배열 기반 컨테이너에서만 작동한다.

```cpp
vector<int> vecInt = {2, 3, 5, 4, 1};

// 오름차순
sort(vecInt.begin(), vecInt.end(), less<int>());     // [1, 2, 3, 4, 5]
sort(vecInt.begin(), vecInt.end(), Less<int>);        // 직접 만든 함수

// 내림차순
sort(vecInt.begin(), vecInt.end(), greater<int>());  // [5, 4, 3, 2, 1]
sort(vecInt.begin(), vecInt.end(), Greater<int>);     // 직접 만든 함수
```

### count_if()
조건자가 `true`를 반환하는 원소의 개수를 반환한다.

```cpp
vector<int> vecInt = {2, 3, 5, 4, 1};

int iCnt = count_if(vecInt.begin(), vecInt.end(), OddNum<int>);
cout << "홀수: " << iCnt << endl;  // 홀수: 3 (1, 3, 5)
```

### for_each()
컨테이너를 순회하면서 조건자를 수행한다.  
동적 할당된 메모리를 해제할 때 주로 사용한다.

```cpp
vector<int*> vecPtr;
vecPtr.reserve(5);
vecPtr.push_back(new int);
vecPtr.push_back(new int);
vecPtr.push_back(new int);

for_each(vecPtr.begin(), vecPtr.end(), Safe_Delete<int*>);  // 메모리 해제
vecPtr.clear();
```

### 조건자 / 템플릿 함수

```cpp
// 오름차순
template<typename T>
bool Less(T _a, T _b) { return _a < _b; }

// 내림차순
template<typename T>
bool Greater(T _a, T _b) { return _a > _b; }

// 홀수 판별
template<typename T>
bool OddNum(T _a) {
    if ((_a % 2) != 0)
        return true;
    return false;
}

// 안전한 동적 메모리 해제
template<typename T>
void Safe_Delete(T& _Dst) {
    if (_Dst) {
        delete _Dst;
        _Dst = nullptr;
    }
}
```

## 9. 컨테이너 선택 기준

| 컨테이너 | 언제 사용? |
|---|---|
| `vector` | 데이터를 많이 **조회**할 때 |
| `list` | 데이터를 많이 **삽입/삭제**할 때 |
| `map` | 데이터를 **쌍(key-value)으로 관리**할 때 |
