# C++ STL list 완전 정리

---

## 1. list란?

각 노드가 **포인터로 연결된 연결 리스트** 컨테이너.  
메모리가 연속되지 않고 여기저기 흩어져 있지만, 포인터로 연결되어 연속된 것처럼 동작한다.

```cpp
#include <list>  // 헤더 추가 필수

list<int> listInt;
```

---

## 2. 원소 삽입

앞/뒤 삽입이 모두 가능하다.

```cpp
listInt.push_back(3);   // {3}
listInt.push_back(4);   // {3, 4}
listInt.push_front(2);  // {2, 3, 4}
listInt.push_front(1);  // {1, 2, 3, 4}
```

---

## 3. 원소 출력

list는 배열 기반 컨테이너가 아니라 **인덱스 접근이 불가능**하다.  
반복자(iterator)를 활용해야 한다.

```cpp
// ❌ 불가능
cout << listInt[i] << endl;

// ✅ 반복자 사용
for (list<int>::iterator it = listInt.begin(); it != listInt.end(); ++it)
    cout << *it << endl;

// ✅ auto 사용 (더 간결)
for (auto it = listInt.begin(); it != listInt.end(); ++it)
    cout << *it << endl;

// ✅ 범위 기반 for (가장 간결, 전체 순회할 때 추천)
for (auto& a : listInt)
    cout << a << endl;
```

---

## 4. auto 키워드 (C++11)

초기화하는 데이터의 타입으로 자동 매칭된다.  
단, **선언과 동시에 초기화**를 진행해야 한다.

```cpp
auto a = 10;     // int
auto b = 3.14f;  // float
auto c = 31.4;   // double
auto d = 'A';    // char
auto e;          // ❌ 오류! 초기화 없으면 타입을 알 수 없음

// iterator에서 활용
auto iter = listInt.begin();
// 위아래 동일한 코드
list<int>::iterator iter = listInt.begin();
```

---

## 5. 범위 기반 for

전체를 순회할 때 사용한다.  
중간에 특정 원소를 건너뛰는 것은 불가능하다.

```cpp
int iArr[5] = {1, 2, 3, 4, 5};

// 기존 방식
for (int i = 0; i < 5; ++i)
    cout << iArr[i] << endl;

// 범위 기반 for
for (int a : iArr)
    cout << a << endl;
```

> ⚠️ `for (int a : iArr)` 는 원소를 **복사**해서 대입한다.  
> 복사 연산을 줄이려면 **`auto&`** (참조)를 사용하자!

```cpp
// ✅ 참조로 순회 (복사 없음, 권장)
for (auto& a : listInt)
    cout << a << endl;
```

## 6. 원소 삭제

```cpp
listInt.pop_back();   // 맨 뒤 원소 삭제
listInt.pop_front();  // 맨 앞 원소 삭제
```

## 7. 반복자 (Iterator)

list는 **양방향 접근 반복자**를 사용한다.  
각 노드가 연속된 메모리에 있지 않기 때문에 임의 접근이 불가능하다.

```cpp
list<int>::iterator iter = listInt.begin();

iter[2];   // ❌ 불가능 (임의 접근 불가)
iter += 2; // ❌ 불가능 (같은 이유)

++iter;    // ✅ 한 칸씩만 이동 가능
--iter;    // ✅ 뒤로도 이동 가능
```

---

## 8. 중간 삽입

```cpp
// listInt 상태: [1, 2, 3, 4, 5]
list<int>::iterator iter = listInt.begin();
++iter;  // 2
++iter;  // 3

listInt.insert(iter, 9);  // iter(3) 앞에 9 삽입

// 결과: [1, 2, 9, 3, 4, 5]
```

### vector vs list 중간 삽입 차이

```
vector 중간 삽입
[1][2][3][4][5]  → 9 삽입 시
[1][2][9][3][4][5]
// 3, 4, 5를 실제로 한 칸씩 복사/이동해야 함
// 원소가 100만 개면 최대 100만 번 복사 발생!

list 중간 삽입
[1]→[2]→[3]→[4]→[5]  → 9 삽입 시
[1]→[2]→[9]→[3]→[4]→[5]
// 포인터 2개만 수정하면 끝!
// 원소가 100만 개여도 포인터 2개만 수정
```

---

## 9. 반복자 무효화

### vector의 경우 (무효화 발생 ⚠️)

```
[1][2][3][4][5]  // 연속 메모리, 주소 100~120
iter가 3을 가리키는 중 (주소 108)

중간에 9 삽입 → 메모리 부족 시 통째로 이동!
[1][2][9][3][4][5]  // 새 주소 200~224

iter는 아직 주소 108을 가리킴 → 쓰레기값 또는 크래시!
```

### list의 경우 (무효화 없음 ✅)

```
[1]→[2]→[3]→[4]→[5]

중간에 9 삽입 → 포인터만 수정
[1]→[2]→[9]→[3]→[4]→[5]

3을 가리키던 iter는 여전히 3을 가리킴 → 문제없음!
```

> 💡 vector는 삽입/삭제 시 메모리를 통째로 재할당할 수 있어서 기존 iterator가 무효화될 수 있다.  
> list는 포인터만 수정하기 때문에 기존 iterator가 여전히 유효하다.

## 10. 중간 삭제

```cpp
// listInt 상태: [1, 2, 3, 4, 5]
list<int>::iterator iter = listInt.begin();
++iter;  // 2
++iter;  // 3

iter = listInt.erase(iter);  // 3 삭제, 삭제 후 다음 원소(4)를 가리키는 iter 반환

// 현재 iter → 4를 가리킴
// 처음부터 다시 순회하려면 명시적으로 재설정 필요
iter = listInt.begin();
```

> ⚠️ `erase()` 후 iter는 삭제된 다음 원소를 가리킨다.  
> 처음부터 순회하려면 `iter = listInt.begin()`으로 재설정해야 한다.

## 11. 정렬

list는 멤버 함수 `sort()`를 사용한다.

```cpp
// listInt 상태: [1, 3, 5, 4, 2]

// 오름차순 (3가지 방법, 결과 동일)
listInt.sort();              // 기본 오름차순 → [1, 2, 3, 4, 5]
listInt.sort(less<int>());   // STL 내장 비교 함수
listInt.sort(Less<int>);     // 직접 만든 템플릿 함수

// 내림차순 (3가지 방법, 결과 동일)
listInt.sort(greater<int>());  // STL 내장 비교 함수 → [5, 4, 3, 2, 1]
listInt.sort(Greater<int>);    // 직접 만든 템플릿 함수

// sort 후 reverse로 내림차순
listInt.sort();     // 오름차순 정렬
listInt.reverse();  // 순서 뒤집기 → 내림차순
```

### 직접 만든 정렬 함수 (템플릿)

```cpp
// 오름차순
template <typename T>
bool Less(T& _a, T& _b) {
    return _a < _b;
}

// 내림차순
template <typename T>
bool Greater(T& _a, T& _b) {
    return _a > _b;
}
```

## 12. 주요 함수 목록

| 함수 | 기능 |
|---|---|
| `push_back(val)` | 맨 뒤에 삽입 |
| `push_front(val)` | 맨 앞에 삽입 |
| `pop_back()` | 맨 뒤 삭제 |
| `pop_front()` | 맨 앞 삭제 |
| `insert(iter, val)` | iter 앞에 삽입 |
| `erase(iter)` | iter가 가리키는 원소 삭제 |
| `begin()` | 첫 번째 원소 반복자 |
| `end()` | 마지막 다음 반복자 |
| `size()` | 원소 개수 |
| `empty()` | 비어있으면 true |
| `clear()` | 모든 원소 삭제 |
| `front()` | 첫 번째 원소 반환 |
| `back()` | 마지막 원소 반환 |
| `sort()` | 정렬 |
| `reverse()` | 순서 뒤집기 |
| `swap()` | 두 list 교환 |

## 13. 컨테이너 선택 기준

| 컨테이너 | 언제 사용? |
|---|---|
| `vector` | 데이터를 많이 **조회**할 때 |
| `list` | 데이터를 많이 **삽입/삭제**할 때 |
| `map` | 데이터를 **쌍(key-value)으로 관리**할 때 |
