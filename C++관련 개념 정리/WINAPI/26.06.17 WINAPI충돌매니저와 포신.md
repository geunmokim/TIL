# WinAPI 충돌매니저와 포신

---

## 1. 내장 수학 함수 (abs, sqrtf)

```cpp
#include <cmath>

abs(-5)     // → 5   (절댓값)
sqrtf(9.f)  // → 3.f (제곱근, float 버전)
```

- `abs` : 음수를 양수로 변환
- `sqrtf` : `sqrt`의 float 버전, 두 점 사이의 거리 계산에 사용

---

## 2. 원형 충돌 판정 (Check_Sphere)

두 오브젝트 사이의 실제 거리(fDia)가 두 반지름의 합(fSumRad)보다 작으면 충돌로 판정한다.

```
          fY
           |  /
           | / fDia (실제 거리, 피타고라스)
           |/__________ fX
```

```cpp
bool CCollisionMgr::Check_Sphere(CObj* _pDst, CObj* _pSrc)
{
    float fX    = abs(_pDst->Get_Info().fX - _pSrc->Get_Info().fX);
    float fY    = _pDst->Get_Info().fY - _pSrc->Get_Info().fY;
    float fDia  = sqrtf(fX * fX + fY * fY);          // 두 중심 사이 거리
    float fSumRad = (float)((_pDst->Get_Info().iCX
                           + _pSrc->Get_Info().iCX) >> 1); // 두 반지름의 합

    if (fSumRad >= fDia)
        return true;   // 충돌
    return false;
}
```

- `>> 1` : 비트 연산으로 ÷2 처리 (iCX + iCX) / 2 = 반지름 합
- `fSumRad >= fDia` : 원이 겹쳤다 → 충돌

## 3. 사각형 충돌 판정 (Collision_Rect)

WinAPI의 `IntersectRect`를 이용해 두 RECT가 겹치는지 판정한다.

```cpp
void CCollisionMgr::Collision_Rect(list<CObj*>& _Dst, list<CObj*>& _Src)
{
    RECT rc = {};
    for (auto& pDst : _Dst)
    {
        for (auto& pSrc : _Src)
        {
            if (IntersectRect(&rc, &pDst->Get_Rect(), &pSrc->Get_Rect()))
            {
                pDst->Set_Dead();
                pSrc->Set_Dead();
            }
        }
    }
}
```

- O(n²) 이중 순회
- `list` 사용 이유 : 매 프레임 죽은 오브젝트 삭제 시 O(1), iterator 무효화 없음

## 4. 도(Degree) → 라디안(Radian) 변환

`cosf`, `sinf` 등 수학 함수는 라디안 단위만 받는다.

```
360도 = 2π 라디안
180도 = π  라디안
 90도 = π/2 라디안

변환 공식: 라디안 = 도 × (PI / 180)
```

```cpp
// 잘못된 사용 - 90라디안으로 인식
cosf(90);

// 올바른 사용 - 90도를 라디안으로 변환
cosf(90 * PI / 180);  // → 0.f
```

## 5. 화면 좌표계와 방향벡터

수학 좌표계와 화면 좌표계는 Y축 방향이 반대다.

```
수학 좌표계         화면 좌표계
    Y↑                  (0,0)──→ X 증가
     |                    |
     +──→ X               ↓ Y 증가
```

포신 방향으로 이동할 때 X에 `-cosf`를 사용하는 이유가 이 때문이다.

```cpp
// 포신 방향으로 이동
m_tInfo.fX += m_fSpeed * -cosf(m_fAngle * (PI / 180));
m_tInfo.fY += m_fSpeed *  sinf(m_fAngle * (PI / 180));
```

## 6. 포신(Barrel) 끝 좌표 계산

플레이어 중심에서 각도 방향으로 특정 거리만큼 이동한 위치가 포신 끝이다.

```
중심(fX, fY)에서 길이(m_iLengthY) 만큼 포신 방향으로 이동

barrelX = fX + (-cosf(angle)) * length
barrelY = fY + ( sinf(angle)) * length
```

```cpp
void CPlayer::Barrel()
{
    m_iBarrelX = m_tInfo.fX + -cosf(m_fAngle * (PI / 180)) * m_iLengthY;
    m_iBarrelY = m_tInfo.fY +  sinf(m_fAngle * (PI / 180)) * m_iLengthY;
}
```

포신 렌더링:
```cpp
void CPlayer::Render(HDC _DC)
{
    MoveToEx(_DC, m_tInfo.fX, m_tInfo.fY, nullptr); // 시작: 플레이어 중심
    LineTo(_DC, m_iBarrelX, m_iBarrelY);             // 끝: 포신 끝
}
```

## 7. 총알 발사 - 포신 방향과 동기화

총알 생성 시 플레이어의 현재 각도를 그대로 넘겨줘서 포신 방향과 동일하게 날아가도록 한다.

```cpp
// 총알 생성 시 각도 전달
CObj* CPlayer::Create_Bullet(BULLET::DIR _eDir)
{
    CObj* pObj = CAbFactory<CBullet>::Create(m_tInfo.fX, m_tInfo.fY);
    static_cast<CBullet*>(pObj)->Set_MyAngle(m_fAngle); // 포신 방향 각도 전달
    return pObj;
}

// 총알 Update - 동일한 방향벡터로 이동
int CBullet::Update()
{
    m_tInfo.fX += m_fSpeed * -cosf(m_fAngle * (PI / 180));
    m_tInfo.fY += m_fSpeed *  sinf(m_fAngle * (PI / 180));
    return OBJ_NOEVENT;
}
```

방향을 제거하면 X, Y 모두 증가 → 오른쪽 대각선 아래로 이동:
```cpp
// 방향 없이 속력만 더하면
m_tInfo.fX += m_fSpeed; // 오른쪽
m_tInfo.fY += m_fSpeed; // 아래쪽 (화면 좌표계)
// 결과: 오른쪽 대각선 아래로 이동
```
