# 모작 제작 과정 — 시야 시스템 적용 및 통합 디버깅

## 1. 시야(Fog of War) 시스템 구현

### 설계 방향
플레이어 시야와 몬스터의 발견 범위는 서로 다른 목적의 별개 시스템이라는 점을 먼저 정리했다. 몬스터의 발견 범위(거리 기반 인지)는 이미 On_Turn() 안에 구현되어 있었고, 새로 만드는 시야 시스템은 오직 "화면에 무엇을 보여줄지"를 결정하는 플레이어 전용 시스템으로 한정했다.

시야 데이터는 맵 전체에 대해 단 하나만 존재하는 상태이므로, TileMgr·TurnMgr와 같은 이유로 싱글톤으로 설계했다. 데이터 저장 방식은 TileMgr가 이미 쓰고 있던 "1차원 벡터 + row*TILEX+col 인덱스 계산" 패턴과 통일했다. 두 벡터(타일 데이터, 시야 데이터)가 같은 인덱스 규칙을 공유하기 때문에, 내용을 복사하지 않고도 같은 칸을 가리킬 수 있다는 점을 직접 검증했다.

### 핵심 코드 — 거리 기반 시야 갱신
```cpp
void CSight::Update_Sight()
{
	GRIDPOS playerPos = CObjMgr::Get_Instance()->Get_Player()->Get_GridPos();

	for (int i = 0; i < TILEY; ++i)
	{
		for (int j = 0; j < TILEX; ++j)
		{
			int index = i * TILEX + j;
			GRIDPOS TilePos(i, j);
			int iDistance = TilePos.Get_Distance(playerPos);

			if (iDistance <= PLAYER_SIGHT_RANGE)
			{
				m_vecVisibility[index] = VISIBLE;
			}
			else
			{
				if (m_vecVisibility[index] == VISIBLE)
					m_vecVisibility[index] = EXPLORED;
			}
		}
	}
}
```

### 렌더링 — GDI AlphaBlend를 이용한 반투명 안개 처리
UNSEEN 상태인 칸은 불투명한 검은 사각형으로, EXPLORED 상태인 칸은 반투명한 안개로 덮어 그리도록 구현했다. GDI의 기본 도형 함수(FillRect)는 반투명을 지원하지 않아, 별도의 임시 메모리 화면에 검은 사각형을 그린 뒤 AlphaBlend로 합성하는 방식을 사용했다. AlphaBlend는 별도 라이브러리(Msimg32.lib)에 있어 링크 에러가 발생했고, 소스 파일에 라이브러리 링크 지시문을 추가해 해결했다.

### 핵심 코드 — 반투명 안개 합성
```cpp
HDC hMemDC = CreateCompatibleDC(_DC);
HBITMAP hBitmap = CreateCompatibleBitmap(_DC, TILECX, TILECY);
HBITMAP hOldBitmap = (HBITMAP)SelectObject(hMemDC, hBitmap);

FillRect(hMemDC, &tLocalRect, hBlackBrush);

BLENDFUNCTION tBlend = {};
tBlend.BlendOp = AC_SRC_OVER;
tBlend.SourceConstantAlpha = 120;

AlphaBlend(_DC, iX, iY, TILECX, TILECY, hMemDC, 0, 0, TILECX, TILECY, tBlend);
```

## 2. 통합 과정에서 발견된 누락 — Add_Object 이중 등록

오브젝트 매니저가 "종류별 분류(OBJID)"와 "렌더링용 분류(RENDERID)"라는 두 개의 별도 리스트를 갖고 있다는 구조를 다시 확인했다. 플레이어/몬스터를 스테이지에 추가할 때 한쪽 리스트에만 등록해 Get_Player()가 빈 리스트를 참조하는 문제가 있었고, 두 리스트 모두에 등록하도록 수정했다.

렌더링용 리스트는 매 프레임 그려진 직후 비워지고, Late_Update() 단계에서 종류별 리스트를 기준으로 다시 채워지는 구조라는 것도 함께 확인했다. 이 구조 자체는 정상적으로 구현되어 있어 손대지 않았다.

## 3. 캐릭터 렌더링 크기 불일치

플레이어와 몬스터가 화면에 전혀 보이지 않는 문제를 단계적으로 디버깅했다. 화면에 디버그 텍스트를 직접 출력해 좌표, 이미지 핸들, 크기값을 하나씩 확인하는 방식으로 원인을 좁혔다.

최종적으로 원본 스프라이트 파일의 실제 크기(플레이어 48x40, 근접 몬스터 28x34, 원거리 몬스터 26x33)와, 그리기 함수에 넘긴 크기(그리드 한 칸 크기인 64x64)가 서로 다르다는 것을 발견했다. 그리기 함수가 원본 비트맵의 실제 영역을 벗어난 부분까지 읽으려 시도하면서 정상적으로 그려지지 못했던 것이다.

그리드 충돌 판정에 쓰이는 크기(m_tInfo.iCX/iCY)와, 화면에 그릴 때 쓰는 원본 스프라이트 크기는 서로 다른 목적의 값이라는 점을 분리해, 몬스터 공통 부모 클래스에 별도의 원본 크기 멤버를 추가하고 각 자식 클래스의 초기화 단계에서 실제 크기를 지정하도록 구조를 정리했다.

### 핵심 코드 — 그리드 크기와 원본 스프라이트 크기의 분리
```cpp
// NewMonster.h
protected:
	int m_iSrcCX;
	int m_iSrcCY;

// Monster_Melee.cpp의 Initialize()
m_tInfo.iCX = TILECX;      // 그리드 충돌 판정용 크기
m_tInfo.iCY = TILECY;
m_iSrcCX = 28;              // 원본 스프라이트의 실제 크기
m_iSrcCY = 34;
```

## 4. 스크롤 보정 누락

캐릭터 크기 문제를 해결한 뒤에도 여전히 위치가 어긋나는 문제가 있었다. 타일을 그리는 코드는 스크롤 값을 화면 좌표에 더해 보정하고 있었지만, 새로 작성한 플레이어·몬스터 클래스의 렌더링 코드에는 이 보정이 빠져 있었다. 좌표 자체(월드 좌표)는 정상이었지만, 화면에 그릴 때 스크롤이 반영되지 않아 스크롤이 진행될수록 타일과 캐릭터의 위치가 점점 어긋나는 결과로 나타났다. 캐릭터의 실제 위치가 변하는 게 아니라, "화면의 어디에 그릴지"를 계산할 때 그 시점의 스크롤 값을 더해야 한다는 점을 다시 확인하고, 플레이어와 몬스터 양쪽의 렌더링 코드에 스크롤 보정을 추가했다.

## 5. 턴 진행 정지 — 매니저 갱신 호출 누락

캐릭터가 보이게 된 이후, 한 번 이동한 뒤로는 키 입력이 더 이상 반응하지 않는 문제가 발생했다. 화면에 현재 턴 상태를 출력해 직접 확인한 결과, 적 턴 상태에서 멈춰 있다는 것을 발견했다. 원인을 추적한 결과, 턴을 관리하는 매니저의 갱신 함수 자체가 스테이지의 매 프레임 갱신 루틴에서 호출되지 않고 있었다는 것을 확인했다. 플레이어가 턴을 종료시키는 호출은 정상적으로 이루어졌지만, 그 상태를 받아 몬스터 턴을 처리하고 다시 플레이어 턴으로 되돌리는 절차 자체가 실행될 기회를 얻지 못했던 것이다. 매니저의 초기화·갱신 호출을 스테이지의 해당 단계에 추가해 해결했다.

## 6. 이동 입력 처리 — 키를 누르고 있는 동안 반복 이동되는 문제

턴 진행이 정상화된 뒤, 이동 속도가 의도보다 훨씬 빠르게 느껴지는 문제를 확인했다. 원인은 매 프레임 호출되는 입력 처리 함수가 "키가 눌려 있는 동안 계속 참"으로 판정하는 방식을 쓰고 있어, 한 번의 입력으로 여러 프레임에 걸쳐 연속으로 이동 판정이 발생하고 있었기 때문이다. 대각선 이동을 지원하기 위해 이 방식을 선택했던 것인데, 그 결과로 "한 번 입력에 한 칸만 이동"한다는 턴제 게임의 기본 전제가 깨지고 있었던 것이다.

해결을 위해 별도의 잠금 상태를 두어, 한 번 이동에 성공하면 즉시 잠그고, 모든 방향키가 떨어지는 순간에만 잠금을 해제하도록 했다. 이렇게 하면 대각선 입력 판정 방식은 그대로 유지하면서도, 키를 누르고 있는 동안에는 한 번만 이동하고 다음 이동을 위해서는 반드시 키를 뗐다가 다시 눌러야 하는 구조가 된다.

### 핵심 코드 — 입력 잠금으로 한 번 입력에 한 칸만 이동하도록 제한
```cpp
if (!bLeft && !bRight && !bUp && !bDown)
	m_bKeyLock = false;

if (m_bKeyLock)
	return;

// ... 방향 판정 및 이동 ...

if (Can_Move(row, col))
{
	Set_GridPos(row, col);
	m_bKeyLock = true;
	// ... 이하 스크롤, 시야 갱신, 턴 종료 처리 ...
}
```
