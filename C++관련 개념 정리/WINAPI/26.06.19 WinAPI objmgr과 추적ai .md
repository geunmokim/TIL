# ObjMgr 공부 노트

---

## 1. static 멤버 변수의 외부 정의

```cpp
// 헤더 (.h)
class CObjMgr {
    static CObjMgr* m_pInstance;  // 선언
};

// 소스 (.cpp)
CObjMgr* CObjMgr::m_pInstance = nullptr;  // 외부 정의
```

- 생성자 초기화 리스트(`: 멤버(값)`)는 인스턴스 멤버 전용 → static에 사용 불가
- static 멤버 변수는 객체가 생기기 전부터 존재해야 하므로 클래스 외부에서 정의 필수
- 특별히 부르는 고유 명칭은 없고 **"static 멤버 변수의 외부 정의"** 라고 칭함

---

## 2. static 멤버 변수 vs 일반 멤버 변수

| | 일반 멤버 | static 멤버 |
|---|---|---|
| 메모리 | 객체마다 생성 | 프로그램 전체에 딱 1개 |
| 초기화 | 생성자에서 가능 | 클래스 외부에서 정의 필수 |
| 생존 기간 | 객체 생명주기 | 프로그램 시작~종료 |
| 핵심 성질 | 각자 소유 | 같은 클래스의 모든 객체가 공유 |

- "공유"는 **해당 클래스에 한정** (다른 클래스의 static과는 무관)
- 클래스명이 네임스페이스 역할 → `CObjMgr::m_pInstance`, `CMonster::s_iCount`
- 전역변수와 비슷하지만 클래스 안에 캡슐화되어 있어 더 안전

## 3. ObjMgr 구조

```
MainGame → ObjMgr → 각 Obj
```

- **MainGame** : 화면 준비, FPS, 화면 지우기 등 담당
- **ObjMgr** : Update, Late_Update, Render, 충돌 등 오브젝트 관련 전담
- MainGame은 호출만 하고 실제 관리는 ObjMgr에 위임하는 구조가 일반적

### 대규모에서는 더 세분화
```
MainGame → ObjMgr    → 각 Obj
                     → RenderMgr    (렌더 전담)
                     → CollisionMgr (충돌 전담)
```

## 4. Set_Target vs Get_Target

### Set_Target (팩토리 방식)
```cpp
CObj* pObj = CAbFactory<CMonster>::Create(pTarget);
// 생성할 때 타겟 고정
// 타겟이 죽어도 포인터를 계속 들고 있음 → 댕글링 포인터 위험
```

### Get_Target (ObjMgr 방식)
```cpp
CObj* pTarget = CObjMgr::Get_Instance()->Get_Target(OBJID::PLAYER, this);
// 매 프레임 살아있는 리스트에서 가장 가까운 대상 탐색
// 타겟이 죽으면 자연스럽게 다음 대상으로 교체
```

| | Set_Target | Get_Target |
|---|---|---|
| 타겟 갱신 | 고정 | 매 프레임 갱신 |
| 댕글링 포인터 위험 | 있음 | 없음 |
| 용도 | 타겟이 바뀔 필요 없을 때 | 추적형 몬스터 등 |

## 5. 댕글링 포인터 (Dangling Pointer)

가리키는 대상이 이미 삭제됐는데 포인터는 그 주소를 그대로 들고 있는 상태

```cpp
CObj* pTarget = pPlayer;
SAFE_DELETE(pPlayer);   // 메모리 해제
pTarget->Update();      // ❌ 이미 삭제된 메모리 접근 → 크래시 or 오작동
```

- 반복자 무효화와 본질적으로 같은 개념
- 둘 다 "가리키던 대상이 사라졌는데 그 주소를 그대로 사용하려 한다"는 문제

### 대규모에서의 해결 방식
```cpp
std::weak_ptr<CObj> m_pTarget;       // 대상 삭제시 자동으로 무효화
TWeakObjectPtr<AActor> TargetActor;  // 언리얼 방식
```

## 6. Get_Target 구현 분석

```cpp
CObj* CObjMgr::Get_Target(OBJID::ID _eID, CObj* _pObj)
{
    if (m_listObj[_eID].empty())
        return nullptr;

    CObj* pTarget = nullptr;
    float fDis = 0.f;

    for (auto& pObj : m_listObj[_eID])
    {
        float fX = pObj->Get_Info().fX - _pObj->Get_Info().fX;  // 가로 거리
        float fY = pObj->Get_Info().fY - _pObj->Get_Info().fY;  // 세로 거리
        float fDia = sqrtf(fX * fX + fY * fY);                  // 실제 직선 거리

        if (fDis > fDia || !pTarget)  // 초기화 트릭 + 최솟값 갱신
        {
            pTarget = pObj;
            fDis = fDia;
        }
    }
    return pTarget;
}
```

- `pObj` : m_listObj[_eID]에서 순회중인 오브젝트 좌표
- `_pObj` : 타겟을 찾으려는 오브젝트(기준점) 좌표
- **수학 공식 + 최솟값 탐색 알고리즘 패턴의 조합**

### 초기화 트릭
```cpp
// !pTarget → 첫 번째 순회시 무조건 true → 첫 오브젝트로 세팅
// 두 번째부터는 fDis > fDia 만 체크 → 더 가까운 오브젝트로 갱신
```

### FLT_MAX 방식 (대안)
```cpp
float fDis = FLT_MAX;  // float 최대값으로 초기화
// !pTarget 없이 첫 번째도 자연스럽게 진입
```

## 7. 몬스터 추적 각도 계산

```cpp
float fX  = m_pTarget->Get_Info().fX - m_tInfo.fX;
float fY  = m_pTarget->Get_Info().fY - m_tInfo.fY;
float fDia = sqrtf(fX * fX + fY * fY);
float fRad = acosf(fX / fDia);  // cos(각도) = 인접변 / 빗변
m_fAngle = fRad * 180.f / PI;

if (m_tInfo.fY < m_pTarget->Get_Info().fY)
    m_fAngle *= -1.f;  // acosf 보정

m_tInfo.fX += cosf(m_fAngle * PI / 180.f) * m_fSpeed;
m_tInfo.fY -= sinf(m_fAngle * PI / 180.f) * m_fSpeed;
```

### acosf의 한계와 보정
- `acosf` 반환값은 **0 ~ 180도**만 가능 → 위아래 구분 불가
- 플레이어가 아래쪽에 있을 때 각도를 음수로 보정하기 위해 fY 비교 사용
- X축 기준으로 짰기 때문에 Y축으로 보정 (반대로 짜도 동작은 동일)

### 가로세로만 이동하게 하려면
```cpp
// 각도 방식 대신 직접 방향 판단
if (m_tInfo.fX < m_pTarget->Get_Info().fX)
    m_tInfo.fX += m_fSpeed;
else
    m_tInfo.fX -= m_fSpeed;

if (m_tInfo.fY < m_pTarget->Get_Info().fY)
    m_tInfo.fY += m_fSpeed;
else
    m_tInfo.fY -= m_fSpeed;
```

## 8. 게임 프로그래밍 삼각함수 정리

### 각 함수의 역할

| 함수 | 역할 | 반환 범위 |
|---|---|---|
| `cosf` | 각도 → X 이동량 분해 | -1 ~ 1 |
| `sinf` | 각도 → Y 이동량 분해 | -1 ~ 1 |
| `acosf` | cos값으로 각도 역산 | 0 ~ 180도 (위아래 구분 불가) |
| `atan2f` | fX, fY로 각도 역산 | -180 ~ 180도 (전 방향 커버) |

### cosf / sinf — 각도를 이동량으로 분해
```cpp
// 각도를 알고 있을 때 X, Y 이동량으로 분해
m_tInfo.fX += cosf(angle) * speed;  // 가로 이동량
m_tInfo.fY -= sinf(angle) * speed;  // 세로 이동량
```
탄막, 부채꼴 발사체처럼 각도를 직접 지정해서 발사할 때 자주 사용

### acosf — cos값으로 각도 역산 (학습용)
```cpp
float fRad = acosf(fX / fDia);  // 0~180도만 반환
// 위아래 구분 못하므로 fY로 수동 보정 필요
if (m_tInfo.fY < m_pTarget->Get_Info().fY)
    m_fAngle *= -1.f;
```
- cos(45도) = cos(-45도) = 0.707 → 같은 값이라 위아래 구분 불가
- 보정 코드가 따로 필요하므로 실무보다는 삼각함수 개념 학습용에 가까움

### atan2f — 실무 표준, 전 방향 커버
```cpp
// fX, fY 둘 다 받아서 부호를 각각 판단
float fRad = atan2f(fY, fX);  // -180 ~ 180도 전부 커버
m_tInfo.fX += cosf(fRad) * m_fSpeed;
m_tInfo.fY -= sinf(fRad) * m_fSpeed;
```
- acosf + fY 보정 코드를 한 줄로 대체
- 실무에서는 acosf 대신 atan2f 사용이 일반적

### atan2f가 acosf를 대체하는 것이지 cosf/sinf까지 대체하는 게 아님
```
각도 구하기  → atan2f (acosf 대체)
이동량 분해  → cosf, sinf (여전히 필요)
```
세 함수가 각자 역할이 있어 함께 쓰임
