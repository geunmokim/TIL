## 1. 추상 팩토리 패턴 (Abstract Factory Pattern)

**정의:** 객체 생성 방법을 한 곳에 몰아넣는 것

**핵심 코드**
```cpp
template <typename T>
class CAbFactory
{
public:
    static CObj* Create()
    {
        CObj* pObj = new T;
        pObj->Initialize();
        return pObj;
    }
    static CObj* Create(float _x, float _y)
    {
        CObj* pObj = new T;
        pObj->Initialize();
        pObj->Set_Pos(_x, _y);
        return pObj;
    }
};
```

**사용 이유**
- 편의성 → template 덕분에 호출 시 한 줄로 끝남
- 유지보수성 → 수정이 필요할 때 팩토리 한 곳만 수정하면 됨

```cpp
// 팩토리 미사용 - 수정 시 전체 수정 필요
CObj* pObj = new CBullet;
pObj->Initialize();
pObj->Set_Pos(x, y);

// 팩토리 사용 - 한 줄로 생성
CAbFactory<CBullet>::Create(x, y);
```

## 2. 게임 제작에서 사용되는 디자인 패턴

| 패턴 | 주 사용처 |
|---|---|
| 팩토리 | 오브젝트 생성 |
| 싱글톤 | 매니저 클래스 |
| 옵저버 | 이벤트 시스템 |
| 상태 | AI, 애니메이션 |
| 컴포넌트 | 오브젝트 기능 조립 |
| 오브젝트 풀 | 대량 오브젝트 성능 |
| 반복자 | 컨테이너 순회 |

### 싱글톤 패턴 (Singleton Pattern)
**정의:** 인스턴스가 딱 하나만 존재하도록 보장하는 패턴

```cpp
class CGameManager
{
public:
    static CGameManager* GetInstance()
    {
        if (m_pInstance == nullptr)
            m_pInstance = new CGameManager;
        return m_pInstance;
    }
private:
    static CGameManager* m_pInstance;
};

// 사용 예시
CGameManager::GetInstance()->Update();
CSoundManager::GetInstance()->Play();
CSceneManager::GetInstance()->Change();
```

### 옵저버 패턴 (Observer Pattern)
**정의:** 이벤트 발생 시 등록된 곳에 자동으로 알려주는 패턴

```cpp
// 플레이어 사망 이벤트 발생 시
// → HP바 UI 갱신, 사망 사운드 재생 등 자동으로 반응
CEventManager::GetInstance()->Subscribe(EVENT::PLAYER_DEAD, UI);
CEventManager::GetInstance()->Subscribe(EVENT::PLAYER_DEAD, SoundManager);
```

### 상태 패턴 (State Pattern)
**정의:** 상태마다 행동을 클래스로 분리하는 패턴

```cpp
class CIdleState   { void Update() { /* 대기 */ } };
class CRunState    { void Update() { /* 달리기 */ } };
class CAttackState { void Update() { /* 공격 */ } };
// if/else 도배 대신 상태 클래스로 분리
// 캐릭터 AI, 애니메이션 상태 관리에 사용
```

### 컴포넌트 패턴 (Component Pattern)
**정의:** 기능을 조립하듯 붙였다 뗐다 할 수 있는 패턴 (유니티의 컴포넌트 시스템)

```cpp
CPlayer* pPlayer = new CPlayer;
pPlayer->AddComponent(new CTransformComponent);  // 위치
pPlayer->AddComponent(new CRenderComponent);     // 렌더링
pPlayer->AddComponent(new CColliderComponent);   // 충돌
```

### 오브젝트 풀 패턴 (Object Pool Pattern)
**정의:** 오브젝트를 미리 만들어놓고 재활용하는 패턴

```cpp
// 총알을 매번 new/delete 하는 대신 재활용
CBulletPool::GetInstance()->GetBullet();    // 꺼내쓰기
CBulletPool::GetInstance()->ReturnBullet(); // 반납
// 총알, 파티클같이 대량 생성/삭제가 반복되는 오브젝트에 필수
```

## 3. 반복자 패턴 (Iterator Pattern)

**정의:** 컨테이너 내부 구조를 몰라도 동일한 방식으로 순회할 수 있게 해주는 패턴

**핵심 코드**
```cpp
// 방식 1 - iterator 직접 사용
auto iter = m_listObj[i].begin();
for (; iter != m_listObj[i].end(); ++iter)
{
    (*iter)->Update();
}

// 방식 2 - 범위 기반 for문 (동일한 반복자 패턴)
for (auto& pObj : m_listObj[i])
    pObj->Render(m_hDC);
```

**장점:** list든 vector든 map이든 내부 구조가 달라도 begin(), end(), ++iter 로 똑같이 순회 가능

```cpp
// list를 vector로 바꿔도 순회 코드는 그대로
vector<CObj*> m_Obj;
list<CObj*>   m_Obj;
// 둘 다 아래 코드로 순회 가능
for (auto& pObj : m_Obj)
    pObj->Render(m_hDC);
```

## 4. 순수 가상 함수 (Pure Virtual Function)

**정의:** 자식 클래스가 반드시 구현하도록 강제하고, 부모 클래스는 직접 생성 못하게 막는 것

```cpp
virtual void Initialize() = 0;  // = 0 → 순수 가상 함수
virtual int Update() = 0;
virtual void Late_Update() = 0;
virtual void Render(HDC _DC) = 0;
virtual void Release() = 0;
```

- `= 0` 이 하나라도 있으면 추상 클래스가 됨
- 추상 클래스는 직접 생성 불가, 자식 클래스를 통해서만 사용 가능

```cpp
CObj* pObj = new CObj;     // 에러! 추상 클래스 직접 생성 불가
CObj* pObj = new CBullet;  // OK! 자식 클래스는 생성 가능
```

## 5. namespace와 enum

**정의:** 이름 충돌 방지를 위해 enum을 namespace로 감싸는 것

```cpp
namespace BULLET  { enum DIR { LEFT, RIGHT, UP, DOWN }; }
namespace MONSTER { enum DIR { LEFT, RIGHT }; }

// 사용 시
BULLET::DIR  m_eDir;   // 총알 방향
MONSTER::DIR m_eDir;   // 몬스터 방향
```

- 타입명 `DIR`, 값 `LEFT`, `RIGHT` 가 겹쳐도 namespace로 구분 가능
- namespace 없이 쓴다면 타입명과 값을 전부 다르게 지어야 함

```cpp
// namespace 없이 쓸 때
enum BULLET_DIR  { B_LEFT, B_RIGHT, B_UP, B_DOWN };
enum MONSTER_DIR { M_LEFT, M_RIGHT };
```

## 6. Update / Late_Update / Render 역할 분리

| 함수 | 역할 |
|---|---|
| Update() | 입력, 이동, 상태 변경 |
| Late_Update() | 충돌 처리, 소멸 판정 |
| Render() | 화면 출력 |

**Late_Update에서 충돌 처리하는 이유**

```
Update()에서 충돌 처리하면
→ 총알 Update() 시점에 몬스터가 아직 이동 전일 수 있음
→ 충돌 판정이 부정확해짐

Late_Update()에서 처리하면
→ 모든 오브젝트 이동 완료 후 최종 위치에서 충돌 체크
→ 정확한 판정 가능
```

## 7. IntersectRect

**정의:** Windows API가 제공하는 두 사각형 충돌 체크 함수

```cpp
IntersectRect(&rec, &pObj->Get_Rect(), &pColObj->Get_Rect())
// &rec                     → 겹친 영역을 저장할 변수
// &pObj->Get_Rect()        → 첫 번째 Rect (총알)
// &pColObj->Get_Rect()     → 두 번째 Rect (몬스터)
// 반환값 → 겹치면 TRUE, 안겹치면 FALSE
```

```
총알 Rect       몬스터 Rect
┌────┐          ┌────┐
│    │    vs    │    │
└────┘          └────┘
  → 안겹침 = FALSE

총알 Rect
┌────┐
│  ┌─┼──┐   ← 겹친 영역이 rec에 저장
└──┼─┘  │
   └────┘
  몬스터 Rect
  → 겹침 = TRUE
```

## 8. 오브젝트 리스트 관리

```cpp
list<CObj*> m_listObj[OBJID::END];  // 종류별 오브젝트 리스트
```

- 오브젝트를 리스트에 등록해야 매 프레임 Update, Render에서 관리 가능
- OBJ_DEAD 반환 시 리스트에서 자동 삭제

```cpp
if (OBJ_DEAD == iEvent)
{
    SAFE_DELETE(*iter);
    iter = m_listObj[i].erase(iter);
}
```

## 9. FPS 측정

```cpp
++m_iFPS;                              // 매 프레임 카운트
if (m_dwTime + 1000 < GetTickCount())  // 1초 경과 시
{
    SetWindowText(g_hWnd, szFPS);      // 타이틀바에 FPS 출력
    m_iFPS = 0;
    m_dwTime = GetTickCount();
}
```

- `m_iFPS` → 1초동안 프레임 카운트
- `m_dwTime` → 1초 측정용 타이머
- `#ifdef _DEBUG` 안에 작성 → 디버그 빌드에서만 동작, 릴리즈 빌드에서 자동 제거
