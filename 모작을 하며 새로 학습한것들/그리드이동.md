# 그리드 이동 (Grid-based Movement)

## 왜 필요했나
턴제 시스템을 구현하려면 캐릭터 위치를 "칸 단위"로 관리해야 했음.
기존 픽셀 단위 이동(Player.cpp)으로는 "한 턴 = 한 칸"이라는 규칙을 표현하기 어려웠음.

## 적용한 방식
- 위치를 GRIDPOS(row, col) 구조체로 따로 관리
- Key_Pressing(연속 입력) 대신 Key_Down(단발 입력)으로 변경
- 기존 Obj 클래스의 INFO(fX, fY)는 그대로 유지하고, 칸 좌표가 바뀔 때 자동 변환
